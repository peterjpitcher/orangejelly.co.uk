## 0. The single most important pattern (the CTR crisis explained)

The site earns **36,241 impressions but only 403 clicks (1.11% CTR) over 16 months**. The blog (`/licensees-guide/*`) generates ~95% of impressions. The problem is not ranking — dozens of pages sit at pos 4-9. The problem is **the click**.

Reading the actual metadata reveals why, and it splits into two failure modes:

1. **Titles are descriptive/brand-led, not query-led.** App pages run through `generateMetadata()` which **appends `" | Orange Jelly"`** to every title, burning ~16 characters of branding and pushing the keyword right. Worse, several titles never contain the exact phrase the page ranks for: `/pub-rescue` ranks for "pub rescue" but is titled "Pub Recovery Support"; `partnering-local-brands...` ranks for "pub co branding" but the phrase is absent; `fizz-street-food-pop-up` ranks for "pop up events for pubs" but the title says "Friday Fizz". Blog "101" titles ("Quiz Night 101", "Cash Bingo 101", "Family Craft Hour 101") use a label nobody searches.

2. **Meta descriptions are feature statements, not benefit + hook + CTA.** The clearest evidence: pages with strong, on-keyword titles **still get 0% CTR** — `buying-a-pub-complete-guide` (pos 5.4, 262 impr, **0 clicks**), `pub-health-check...` ("how to run a pub", pos 7.0, 0.43%), `cellar-management...` (pos 6.5, 0.51%), and the Instagram/Facebook service pages (**pos 2.1-2.4, 0 clicks**). When the title is right and clicks are still zero, the description is the lever. Blog descriptions fall back to the `excerpt`, written as a summary ("Proven pub food menu ideas that balance customer appeal with profitability...") rather than a reason to click.

**The fix is a repeatable two-part on-page pattern, applied in priority order:** (a) front-load the exact GSC query phrase in the title, drop filler and redundant branding; (b) rewrite the meta description to lead with the specific benefit/result, add a concrete proof point (food GP 58%->71%, quiz to 35 regulars), and end with a soft CTA. The infrastructure already supports this cleanly — `seo-overrides.ts` (104 entries) lets you set title+description per URL without touching page bodies.

### Internal-linking plan (the equity-funnelling job)
The four mega-pages — **summer-pub-event-ideas (6427), profitable-pub-food-menu-ideas (4137), social-media-strategy-for-pubs (3601), quiz-night-ideas (3000)** — hold ~13,000 of the site's 36,000 impressions but are informational dead-ends. Each must carry **clear, contextual links to the matching commercial page**:

- `social-media-strategy-for-pubs` + `content-marketing-ideas-pubs` + `instagram/facebook-marketing` -> **`/services/*`** (the pos-2 pages that get no clicks — feed them internal traffic too).
- `summer-pub-event-ideas` / events pillar -> **`/quiet-midweek-solutions`** and **`/empty-pub-solutions`** ("got the ideas but no time? we'll run them").
- `profitable-pub-food-menu-ideas` / `quiz-night-ideas` -> **`/ways-to-work`** package pages and **`/results`** (proof).
- `why-is-my-pub-empty` / turnaround cluster -> **`/fix-my-pub`** and **`/pub-rescue`**.
- All commercial pillars -> **`/results`** (proof) and **location pages** where geographically relevant.

Mechanism: contextual in-body links + an end-of-article "Want this done for you?" CTA block pointing to the cluster's commercial endpoint. This bridges the blog's impression volume into commercial-page authority and leads.

### Sequencing (feeds the master synthesis)
1. **Week 1 — CTR rescue (title+meta only, via `seo-overrides.ts`):** the 5 `/services/*` pages (pos 2-3, 0 clicks), `buying-a-pub-complete-guide`, `pub-health-check`, `cellar-management`, `/pub-rescue`, `partnering-local-brands`. Fastest ROI, no content writing.
2. **Week 2 — commercial pillar + cannibalisation:** fix `/pub-marketing` framing; resolve quiz, empty-tables, chains, local-marketing merges.
3. **Week 3+ — gap content:** pop-up events + family events rebuilds; refurb pillar depth.
4. **Ongoing — internal linking** rollout across the four mega-pages, plus indexing the 23 zero-vis posts.
