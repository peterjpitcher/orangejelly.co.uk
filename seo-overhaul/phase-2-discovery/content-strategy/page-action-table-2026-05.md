# Per-Page Action Table — Orange Jelly (2026-05-21)

Built from the merged GSC inventory (`gsc-baseline-2026-05.md`, 117 pages with visibility) cross-referenced against the live codebase (`src/app/**/page.tsx` routes + all 96 `content/blog/*.md` posts). Pages with zero GSC visibility are included so the worklist is complete.

## How to read this
- **16mo impr / pos**: from GSC. `—` = no GSC visibility (too new / not indexed / never surfaced).
- **Title problem**: assessed by reading the ACTUAL metadata. App pages run through `src/lib/metadata.ts` `generateMetadata()` which **appends `" | Orange Jelly"`** to every title (≈16 chars of branding). `src/lib/seo-overrides.ts` (104 entries) overrides title+description for most blog posts and several app pages. Blog titles do NOT get the suffix (the `[slug]` route returns `resolvedTitle = override?.title || metaTitle || post.title` with no template); blog meta description falls back to the post `excerpt`.
- **Verdict**: Optimise / Rewrite title+meta / Rewrite content / Merge→[target] / Leave / Remove/Noindex / New.
- **Priority**: P1 = do first (high impr × striking distance × commercial value); P2 = next; P3 = later/low-value.

## THE CTR-CRISIS PATTERN (applies site-wide)
Pages rank but titles are **descriptive/brand-led, not query-led**, and meta descriptions are **feature statements, not benefit + hook + CTA**. The searcher's exact phrase (e.g. "event ideas for pubs", "instagram services for pubs", "pub menu ideas") rarely appears verbatim near the front of the title. Net effect: pos 5-12 with 0-1% CTR. Default on-page action for almost every ranking page = **front-load the exact query phrase in the title and rewrite the meta description to lead with the benefit + a soft CTA**, then improve internal links/freshness.
