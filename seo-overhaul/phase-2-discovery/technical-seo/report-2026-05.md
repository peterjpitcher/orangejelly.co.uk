# Technical SEO Audit — Orange Jelly (orangejelly.co.uk)

**Date:** 2026-05-21 | **Auditor:** Technical SEO Specialist | **Type:** Planning / audit only (no code or indexation changes made)
**Stack:** Next.js 15 App Router (React 19, TS), Vercel | **Codebase root:** `/Users/peterpitcher/Cursor/OJ-OrangeJelly.co.uk`
**Method:** Direct codebase reads + live `curl` (status, headers, rendered HTML, JSON-LD), GSC baseline cross-reference. PageSpeed Insights API was rate-limited (no key) — CWV is a directional code-based assessment, explicitly flagged.

> **Headline:** The site's technical foundations are genuinely strong (self-canonical, rich JSON-LD, HSTS+CSP, valid sitemap, correct robots). The CTR/visibility crisis in GSC is **NOT** primarily a crawl/index problem. It is driven by (1) the April overhaul **301/308-redirecting four service sub-pages that still rank well**, throwing away ranking equity onto a generic page; (2) **weak internal link equity** to commercial pages from an authority-poor hub; and (3) **brand-new, zero-authority location pages with negligible search demand**. Most "anomalies" in the brief are already-correct fixes (e.g. `/about-demo` is a 410) whose GSC rows are stale 16-month residue.

## 1. Critical Issues (fix first)

### C1 — Four well-ranking `/services/*` sub-pages are 308-redirected to a generic page, destroying earned rankings
- **Evidence:**
  - `next.config.js` `redirects()` 301s (`permanent: true`, served as 308) `/services/instagram-services-for-pubs`, `/services/facebook-services-for-pubs`, `/services/paid-social-for-pubs`, `/services/content-creation-for-pubs`, `/services/social-media-marketing-for-pubs` → `/capabilities`, and `/services` → `/ways-to-work`.
  - Live: `curl -sI /services/instagram-services-for-pubs` → `HTTP/2 308, location: /capabilities`. Same for facebook/social-media/services.
  - GSC: these URLs rank **pos 2.1–6.5** with impressions (instagram 32 impr pos 2.4; facebook 33 impr pos 2.1; paid-social 30 impr pos 1.9; content-creation 23 impr pos 2.7). Queries "instagram services for pubs" pos 6.5 (236 impr), "facebook services for pubs" pos 5.8 (103 impr), "paid social for pubs" pos 11.2 (186 impr), "content creation for pubs" pos 14.8 (211 impr) — all **0 clicks**.
  - Conflict: `src/app/services/instagram-services-for-pubs/page.tsx` calls `permanentRedirect('/services/social-media-marketing-for-pubs')` — but that target *also* config-redirects to `/capabilities`. Layered/contradictory redirect intent; the config rule wins.
- **Root cause:** The April overhaul collapsed dedicated, query-matched service pages into one generic `/capabilities` page via 308. Google still serves the old URLs (cached rankings), but every click now redirects to a page that does **not** match the specific query ("instagram services for pubs" → a generic capabilities list). Result: ranked, zero clicks, and the specific ranking equity is being consolidated onto a page that cannot satisfy the intent. Over time Google will drop these positions because the redirect target is a topical mismatch.
- **Severity:** Critical | **Confidence:** High | **Impact:** Commercial bottom-funnel queries | **Effort:** M | **Dependencies:** Content/strategy decision on whether to revive dedicated pages.
- **Recommended action:** **Reinstate thin-but-focused dedicated pages** at the four ranking `/services/*` slugs (Instagram, Facebook, paid-social, content-creation), each self-canonical and targeting its exact query, with a clear CTA to `/ways-to-work`. If the business genuinely wants one page, point the 301 to the *most relevant* section anchor and keep query-matching H1/copy on the destination — but a dedicated page will recover the pos 2–6 rankings far better. Keep `/services` → `/ways-to-work`.
- **Validation:** After deploy, `curl -sI` returns 200 with self-canonical; GSC URL Inspection shows "indexed"; watch clicks on those four queries.
- **Risk/rollback:** Low — re-adding pages is additive. Rollback = restore the redirect.

### C2 — `/test-shadcn` is a live, public, internal component-test page
- **Evidence:** `src/app/test-shadcn/page.tsx` exists; live `curl -sI /test-shadcn` → `HTTP/2 200`. It DOES set `noIndex: true` via `generateStaticMetadata` (renders `<meta name="robots">`, not an HTTP header — hence absent from `curl -sI`). Not in sitemap. Orphan (grep: not linked anywhere). GSC lists it under "no visibility."
- **Root cause:** Leftover from the shadcn migration (created Aug 2025).
- **Severity:** High (low SEO harm, but a public internal page is unprofessional and wastes crawl) | **Confidence:** High | **Effort:** XS | **System-level.**
- **Recommended action:** Delete `src/app/test-shadcn/` entirely. (Noindex already protects rankings; deletion removes the public surface.)
- **Validation:** `curl -sI /test-shadcn` → 404. **Risk:** None (orphan).

### C3 — Organization / LocalBusiness JSON-LD references a non-existent logo URL
- **Evidence:** `EnhancedBlogSchema.tsx` publisher logo = `${baseUrl}/images/logo.png`; `PubMarketingLocationLandingPage.tsx` `LocationLocalBusinessSchema` `image` = `${baseUrl}/images/logo.png`. File check: **`/public/images/logo.png` MISSING** — the logo lives at `/public/logo.png` (and `/logo.png` is in GSC). So the `logo`/`image` URL in structured data 404s across **all ~98 blog posts and all location pages**.
- **Root cause:** Path typo / asset moved without updating schema builders.
- **Severity:** High | **Confidence:** High | **Impact:** Organization entity, Article/LocalBusiness rich-result eligibility, Knowledge Panel | **Effort:** XS | **System-level (2 files).**
- **Recommended action:** Point both to the real asset (`/logo.png`) or add `public/images/logo.png`. Standardise on one canonical logo path used everywhere.
- **Validation:** Rich Results Test on a blog URL shows a resolvable logo; `curl -I` the logo URL → 200.

## 3. Sitemap vs Reality

**Live sitemap:** 132 URLs (`curl /sitemap.xml`). Generated by `src/app/sitemap.ts` (static + marketing routes + dynamic blog + categories). Excludes `/about-demo`, `/test-shadcn`, `/services/*` — correct.

**In sitemap but NOT in GSC (zero impressions) — investigated:**
- `/pub-marketing-agency`, `/pub-marketing-no-budget`, `/pub-marketing-london`, `/pub-marketing-surrey`, `/pub-marketing-hampshire`, `/pub-marketing-hertfordshire`, `/pub-marketing-buckinghamshire`, `/pub-marketing-oxfordshire` — **all live 200, self-canonical, index/follow, in sitemap, rich schema, internally linked from `/pub-marketing`.** Directories created **Apr 5 2026** (≈6 weeks ago).
  - **Root cause = NOT technical.** Three compounding factors: (a) **too new** — barely a crawl cycle since launch; (b) **near-zero search demand** for "pub marketing [county]" (no such queries appear in GSC; even indexed siblings `pub-marketing-kent` 28 impr / `-berkshire` 35 impr / national "pub marketing agency" only 303 impr/16mo); (c) **no domain authority + thinnest possible inbound linking** (only inbound is `/pub-marketing`, itself at 2 impressions / pos 2). They are indexable but have nothing to rank *for* yet. `/pub-marketing-agency` is the exception worth pushing (303 impr of latent demand at pos 19.4).
- **Action:** Keep indexable; do **not** expect organic from county pages short-term. Prioritise `/pub-marketing-agency` (real demand). Treat county pages as low-priority/long-tail or PPC landing pages. Differentiate copy (currently a shared `PubMarketingLocationLandingPage` template fed by per-county JSON — risk of near-duplicate thin content if county JSON is boilerplate; verify content uniqueness).

**Indexed/ranking but NOT in current sitemap (orphans in reporting):**
- `/services/*` sub-pages (now 308 → see C1), `/about-demo` (410), `/test-shadcn` (noindex). Also cross-subdomain rows (see §7). All explained — no true orphan content pages found.

**Sitemap `lastModified` accuracy:** Static/marketing pages use **hardcoded** dates (`2026-04-05`, `2026-03-17`). Blog uses real `updatedDate || publishedDate` (good). Hardcoded dates risk going stale and misleading crawl scheduling. **Low.** Action: derive marketing-page dates from content/git where practical.
